 Crewlyze Enterprise Technical Manual

        
      

      
      
      

        
        
        

          

            
          

          
          
          
          
          
          
          
          
          
          
          
        


        
        

          
          
          

            
 Platform Vision & Architecture

            

              
Crewlyze is an **Autonomous Multi-Agent Business Intelligence and Data Engineering Platform**. Unlike traditional static tools that run scripted sequence statements, Crewlyze models a collaborative human data department using role-specific AI agents that share context, self-correct errors, and execute sandboxed code.

              

                
Core Features:

                
    
                  
  • Multi-Agent Swarm Orchestration: Built using CrewAI for agent scheduling and tasks execution.
    • 
                  
  • Dynamic Delimiter Auto-Healing: Intelligently handles diverse formats (commas, semicolons, tabs) without user configuration.
    • 
                  
  • Natural Language-to-SQL Compiler: Talk directly to your SQLite database tables.
    • 
                  
  • Outbound Webhook Channels: Dispatch reports dynamically to SMTP Email, Slack, Discord, and Custom REST Endpoints.
    • 
                

              

            

          


          
          

            
 CrewAI Swarm & Agent Personas

            

              
The platform assigns distinct, cooperative personas to execute your data analysis tasks:

              
              

                

                  1. Data Quality & Profiling Agent (cleaner.py)
                  
Evaluates dataset structural health. It drops columns containing more than 60% missing values, infers standard types, standardizes headers (removing spaces/special symbols), and imputes numeric empty fields (NaN) with means/medians based on standard deviation skewness. Writes a structured cleaning log detailing changes.

                

                
                

                  2. Relationship & Correlation Analyst (relation.py)
                  
Maps bivariate and multivariable data relationships. Calculates Pearson linear indices and Spearman rank correlations (detecting non-linear curves). Also runs analysis of variance (ANOVA) tests on categorical attributes to locate statistically significant drivers.

                

                
                

                  3. McKinsey Strategy Consultant (insights.py)
                  
Translates raw numerical correlations and statistics into strategic action. Generates SWOT matrices, highlights critical operational bottleneck risks, models growth ROI, and writes C-suite action checklists.

                

                
                

                  4. Plotly Chart Engineer (visualizer.py)
                  
Reads dataset metadata and dynamically compiles Python scripts to build responsive Plotly visualizations. It outputs HTML/SVG chart snippets rendered directly on your UI dashboard.

                

              

            

          


          
          

            
 Ingestion Engines & Analysis Modes

            

              
Crewlyze supports multi-format file parser pipelines and multiple processing speeds:

              
              

                

                  Fast (Standard) Execution Mode
                  
Optimized for speed. Agents run single-turn prompts with fewer structural reflections. It skips complex nested analyses, providing rapid sanitization logs and standard chart dashboards in seconds.

                

                
                

                  Deep Execution Mode
                  
Forces multi-pass reasoning. Agents evaluate dataset edge-cases, perform extensive outlier checks, audit multivariable interactions, and execute self-correcting logic on visual outputs to draft highly detailed reports.

                

                
                

                  Ingestion & Delimiter Healing
                  
On CSV uploads, the backend analyzes the first 50 lines to detect separators (`,`, `;`, `\t`, `|`). If Excel workbooks have multiple sheets, a tab picker allows selecting the target. Ingesting SQLite database files extracts schemas and lets the user choose target tables dynamically.

                

              

            

          


          
          

            
 Chat AI & SQL Copilot

            

              
Interact with your dataset dynamically using our conversational SQL copilot:

              
    
                
  • NLP-to-SQL Compilation: Ask questions in plain English (e.g. "What is the average transaction total grouped by region?"). The copilot inspects the schema and compiles a corresponding SQL query.
    • 
                
  • Dynamic Self-Healing Loop: If the compiled SQL fails (due to join errors or column mismatches), the copilot catches the database exception traceback, feeds it back to the compiler prompt, and automatically rewrites the query on the fly.
    • 
                
  • Dynamic Visualization Engine: The copilot detects if query outputs (like aggregates) are visual-friendly and renders a corresponding Plotly chart in the chat thread.
    • 
              

            

          


          
          

            
 Python Sandboxing & Security

            

              
To safely run AI-generated data code, Crewlyze implements strict sandboxing architectures:

              
    
                
  • Isolated Subprocess Execution: Pandas cleaning code and Plotly chart scripts run in separate subprocess containers, protecting the main server thread.
    • 
                
  • Import Sanitization Guards: Scans and blocks code templates containing system calls (`os.system`), shell escapes, filesystem deletions (`shutil`), or outbound network calls (`urllib`, `requests`).
    • 
                
  • Local Directory Layout: Project workspace directories, original tables, and reports are saved locally at `~/.crewlyze/data/sessions/` without cloud storage.
    • 
              

            

          


          
          

            
 Universal LLM Gateways

            

              
Crewlyze supports 100+ LLMs via the LiteLLM gateway. Configure providers in the Settings Modal:

              
    
                
  • Cloud APIs (OpenAI, Anthropic, Gemini, Groq, Together): Input your secure API credentials and enable Show in sidebar. Recommended models are gpt-4o, claude-3-5-sonnet, or gemini/gemini-2.5-flash.
    • 
                
  • Local Offline Execution (Ollama): Boot Ollama on your machine (e.g. `ollama run llama3.1`), configure base URL to `http://localhost:11434`, and select model ID matching Ollama registry (e.g. `ollama/llama3.1`). Keep keys blank.
    • 
                
  • Custom Enterprise Proxies: Use the `custom` provider to route requests to custom, OpenAI-compatible proxy gateways using endpoint URLs and Bearer headers.
    • 
              

            

          


          
          

            
 Cooldowns & 429 Backoff

            

              
Heavy agentic workloads generate rapid consecutive API requests, which often trigger rate limits from cloud LLM hosts:

              
    
                
  • 429 Rate Limits: Triggered when token-per-minute (TPM) or request-per-minute (RPM) limits are exceeded.
    • 
                
  • API Cooldown Delay: In General Settings, set the Cooldown timer (1s to 60s). This forces a time buffer between consecutive agent prompts, letting rate limits reset.
    • 
                
  • Mathematical Cooldown Backoff: When the backend receives a `429` status, it executes an automated retry loop, sleeping for `cooldown * retry_attempt` seconds to recover safely.
    • 
              

            

          


          
          

            
 Outbound Integrations Hub

            

              
Configure automated dispatch channels in the Integrations tab:

              
    
                
  • SMTP Email client: Provide relay parameters (SMTP host, port: 587 STARTTLS / 465 SSL, credentials). Delivers email summaries and attached PDF reports.
    • 
                
  • Slack Webhooks: Paste channel Webhook URLs. Dispatches key metric summaries directly to your Slack channels.
    • 
                
  • Discord Webhooks: Delivers embeds featuring customized border styles, warning metrics, and attached PDF reports.
    • 
                
  • REST Endpoint Webhooks: POSTs JSON metadata payloads and report files to your custom API endpoints.
    • 
              

            

          


          
          

            
 Export Formats & ZIP Portability

            

              
Crewlyze packages project assets in multiple formats for migration and reporting:

              
    
                
  • PDF Executive Report: Renders a structured ReportLab PDF complete with headers, footers, pagination, SWOT matrices, risk indices, and high-res chart images.
    • 
                
  • Jupyter Notebook (.ipynb): Exports a complete Jupyter notebook package containing pandas configurations and Plotly visualization code to re-run locally.
    • 
                
  • ZIP Project Backup: Bundles the configuration settings, the original dataset, the cleaned dataset, charts, logs, and generated reports into a portable archive.
    • 
              

            

          


          
          

            
 FAQ & Troubleshooting

            

              
    
                
  • Issue: Stale/Orphaned running states: If page reloads occur during active pipeline execution, the backend recovery hook runs on load, scans the directory, terminates zombie subprocesses, and safely resets records to "failed".
    • 
                
  • Issue: FastAPI CORS Block Errors: Ensure remote domains are whitelisted in FastAPI's `CORSMiddleware` in `main.py` if hosting the backend on a remote instance.
    • 
                
  • Issue: Large Dataset Crashes (1GB+): Ingestion is done in-memory via pandas. For massive datasets, verify that your server is configured with sufficient swap memory, or pre-filter datasets.
    •
